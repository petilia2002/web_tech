КРАТКО, ЧТО СДЕЛАНО

PostgreSQL: база `webapp_db`, схема `auth` с таблицами `users`, `roles`, `user_roles` (многие-ко-многим).
Backend: FastAPI (`main.py`) с REST-CRUD по пользователям и ролям + выдача/отзыв роли пользователю.
Frontend (статические страницы):
    `users_menu.html` (список пользователей: поиск/сортировка/пагинация),
    `edit_user.html` (просмотр/редактирование/удаление, создание при `?create=1`, блок «Роли пользователя»),
    `roles_menu.html` (список ролей),
    `edit_role.html` (просмотр/редактирование/удаление, создание при `?create=1`).

—
ЧТО НУЖНО ДЛЯ ЗАПУСКА

1. PostgreSQL запущен, база `webapp_db` создана. В БД выполнены скрипты создания схемы `auth` и таблиц (`users`, `roles`, `user_roles`).

2. В корне проекта файл `.env`:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=webapp_db
DB_USER=postgres
DB_PASSWORD=your_password
```

3. Установка зависимостей (в вирт. окружении):

```
pip install -r requirements.txt
```

4. Запуск сервера (любой из вариантов):

```
python main.py
# или
uvicorn main:app --reload
```

—
КУДА ЗАХОДИТЬ В БРАУЗЕРЕ

    Меню пользователей: `http://127.0.0.1:8000/`
  (кнопки: «Открыть форму по ID», «Создать пользователя», переход в «Меню ролей»)
    Форма пользователя: `http://127.0.0.1:8000/edit`
  (редактирование/удаление по ID; создание при `?create=1`; блок «Роли пользователя» — выдать/отозвать роль)
    Меню ролей: `http://127.0.0.1:8000/roles`
    Форма роли: `http://127.0.0.1:8000/role-edit`
  (редактирование/удаление по ID; создание при `?create=1`)
